Cockpit launcher architecture · v0.6 · 2026-05-12

One click on the desktop shortcut takes the Cockpit from cold to listening. The launcher is the orchestration layer: it checks for an existing instance, wakes Roon Bridge if it's asleep, kicks the QNAP container if SSH is configured, starts Flask, primes the bootstrap endpoint, and opens a Chrome --app window. Every subprocess runs silent. The browser opens with live state on the first frame.

Cold-start flow

Reuse path (singleton hit) Recovery path (Bridge or Server wake) Boot path (Flask, wait, primed) UI path (Chrome --app)

Step contract

StepActionFailure mode + handling
Step 1 Singleton: HTTP probe to /api/config If Cockpit responds with our signature, open Chrome and exit. If port is held but no response, fall through to Flask boot (will fail to bind — manual recovery via Task Manager).
Step 2 Bridge wake: probe RAATServer.exe, spawn Bridge.exe if absent If none of four candidate paths exist, return not_installed and continue silently. Cockpit's in-UI pill surfaces the same conclusion after Flask boots.
Step 3 Server wake: SSH inspect, docker start if down If config.roon_server.ssh_key_path is empty or the key file is missing, skip with no_key. Cockpit's reachability fallback covers detection.
Step 4 Flask: daemon thread, port 5000 If wait_for_port times out after 15s, exit 1. Common cause: zombie pythonw holding the port. Manual recovery via Task Manager.
Step 5 Warm bootstrap: /api/system/bootstrap Response discarded; the effect is forcing every downstream probe to complete before the browser opens. Failure: silent — the polling cadence catches up within ~8s of UI load.
Step 6 Chrome --app window Walks Chrome x64, x86, LOCALAPPDATA, Edge x64, x86. Falls back to webbrowser.open() if all miss. UI works either way; chromeless window is cosmetic.

Operational principles applied

PrincipleHow the launcher honors it
Diagnostics before changesEvery write (spawn Bridge, start container, start Flask) is preceded by the corresponding read (tasklist, inspect, port probe).
Read-only before writetasklist, sc query, docker inspect, port probe — all read-only and cheap.
Minimal scopeNo registry edits, no service stop calls from the launcher, no admin elevation, no policy changes.
ReversibleBridge spawn closes via tray. Container start reverses with docker stop. Flask exits when launcher exits. Chrome window closes via window chrome.
Silent UXCREATE_NO_WINDOW on every subprocess. Zero console flashes.
Fail openEvery pre-flight fallback returns a state code, never raises. Cockpit's in-UI bootstrap is the canonical truth.
